Context#

The S-CORE v1.0 roadmap is structured around four release gates. Each gate has mandatory minimum deliverables; teams are free to advance ahead but the gate requirements are non-negotiable for the final release.

Hardening & Release

v0.8 is the baseline for the Time module. The actual v0.8 scope for TIM is intentionally narrow: the S-CORE project gate for v0.8 calls for requirements and architecture complete, but the Time module is entering that gate with a different priority — stabilise the public API, confirm the end-to-end functional chain, and integrate into reference_integration CI (#68 is the only sub-issue under the v0.8 PI #69). Existing general documentation (README, examples, informal PlantUML diagrams) and the three pre-existing comp_req sets (time, time_daemon, time_slave) are carried over from prior work but are not formal v0.8 deliverables.

This roadmap describes a realistic step-by-step plan that achieves all v1.0 gate requirements by December 2026, treating v0.9 as the primary catch-up sprint for process artefacts.

Feature scope for v1.0 — features marked deferred have no valid_from ≤ v1.0 on their feat_req nodes and are automatically excluded from v1.0 coverage statistics; their component artefacts (comp_req, comp_arc, comp_dd_sta) will be authored in a post-v1.0 PI.
